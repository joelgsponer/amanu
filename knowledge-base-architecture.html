What it is

Two memories, not one

The idea that makes it last is splitting memory in two — one for what your documents say, one for how you like things done.

Memory · the matter

The knowledge base

A wiki-style store over your documents: immutable sources plus a synthesised, cross-linked layer the agent keeps current. The single source of truth for what you know.

Memory · the method

Agent memory

The agent's own durable preferences and lessons — one fact per file, recalled when relevant. It shapes how every task is done, and compounds the longer you use it.

It's deliberately low-tech: plain Markdown, folders, a little YAML, and wiki-links. No database, no server, no lock-in — human-readable and yours, on your machine.

How it's structured

Three layers, plain files

Sources never change. The synthesis is freely rewritten. A single schema file is the constitution that ties them together.

your-knowledge-base/ ├─ SCHEMA.md Layer 3 — the constitution (conventions & workflows) ├─ AGENTS.md · CLAUDE.md auto-loaded entrypoints that orient any agent ├─ inbox/ drop zone for new documents ├─ raw/ Layer 1 — immutable source files, by category ├─ kb/ Layer 2 — the synthesis the agent owns │ ├─ entities/ topics/ sources/ queries/ │ └─ index.md · log.md · overview.md ├─ memory/ the portable brain — a linked graph of conventions, gotchas, solutions (+ index.md) ├─ tools/ self-arming toolbox (+ index.md: purpose · start/stop/status) └─ daily/ session logs that feed the self-arming loop

Every page carries YAML frontmatter (so the store is queryable) and links related pages with [[wiki-links]] (so it's a navigable graph). Layer 3, SCHEMA.md, is the constitution — read it and you can run the whole system. memory/ and tools/ are the agent's brain and toolbox — more on those next.

What the agent does

Your tireless librarian

Five core commands do the work — each a one-word command once the system is built.

ingest

File & cross-reference

Read a new document, classify it, move it into raw/, write a source page, and roll the facts up into the entities and topics it touches.

query

Answer from the synthesis

Answer a question from the kb/ pages — citing them — instead of re-reading raw files. Save anything worth keeping.

lint

Keep it honest

Sweep for contradictions, stale claims, orphan pages, and gaps, and report what needs a human's eye.

compile

Grow its own brain

Distil the daily/ observations into memory/ facts and reusable tools/ scripts — the self-arming loop, on demand.

healthcheck

Audit itself

Verify the whole running system — manifest vs. reality, services, secrets, link integrity — and write a single HTML report you can read.

And it documents itself as it goes: a CHANGELOG.log records every decision and why; a kb/log.md tracks routine operations; and it writes CLAUDE.md + AGENTS.md so any future agent that opens the folder is instantly oriented.

It also arms itself over time: as it works it jots observations to daily/, and the core /compile distils those into the memory/ brain — and, when it spots a recurring task, into a reusable script in tools/. This runs on demand by default (and automatically once you add the Automated tier), so the system you build keeps getting better at your work.

The memory system

A brain that travels with the folder

amanu splits memory by one question: should this survive moving the folder to another machine? Everything about how to work this knowledge base lives inside it; only machine-specific facts stay behind. That's what makes the whole thing portable.

in the folder · portable

memory/ + tools/ — the brain

A cross-linked graph of conventions, gotchas, and solutions the agent works out — like the kb, but tuned for method — plus a self-arming toolbox of scripts. This is the agent's method, and it travels with the folder.

in the folder · portable

kb/ — the matter

Your documents' synthesis: the durable record of what they say, cross-referenced and kept current. Part of the same portable folder.

Machine memory · stays putThe harness's own memory (e.g. ~/.claude/…) holds only machine-specific facts — "this box needs the VPN", "on this Mac do it this way", local install paths. It belongs to the machine, not the folder.

The payoff — portability. Move the folder to a new machine and the agent behaves the same, because its brain and tools came along; you only re-teach the handful of machine-specific things. The auto-loaded AGENTS.md is what lifts all this into each session — so keep harness auto-loading on.

A graph, not a pile. The brain is cross-linked like the kb but tuned for method: each fact is typed (convention, gotcha, solution, preference, procedure, tool) and linked to related facts, the kb pages it concerns (applies-to), and the tool that implements it (implemented-by) — so method, matter, and toolbox form one navigable graph. With the obsidian-vault extension you can browse it visually: a combined vault renders the method graph and the document graph together, with the cross-links between them — and colour-coded so entities, topics, sources, memory facts and tools stand out at a glance.

Self-arming & run-state. The brain and toolbox grow as you use the system: a periodic pass mines the daily/ logs for repeated work and distils it into linked memory/ facts and tools/ scripts. And background tools (watchers, schedulers) are started and registered when built, their state tracked in tools/index.md — so the agent can tell you at a glance what's running and what isn't.

How you set it up

Build it with an agent

You don't configure this by hand. You hand the agent one file and it builds the system with you — a guided, eleven-phase co-build that is the design, the onboarding, and the tutorial all at once.

P0 orientP1 interviewP2 scaffoldP3 schemaP4 seed memoryP5 first ingestP6 first queryP7 skillsP8 extensionsP9 lint + health + docsP10 auto health-check + handoff

Every phase runs the same honest rhythm — and stops leave the system working:

teach→tailor→build→verify→checkpoint→log

It's resumable (a machine-readable amanu.yaml manifest tracks where you are), grows in tiers (Starter → Connected → Automated → Intelligent), and handles secrets safely (.gitignore first, then a .env.example — values never committed).

Privacy & principles

Yours, and trustworthy

  • Two memories, kept apart. Method in agent memory, matter in the knowledge base.
  • Sources are immutable. The synthesis is freely rewritten; the originals never change.
  • Everything is plain files. Markdown, frontmatter, folders, version control — no database, nothing you can't open by hand.
  • Observe before interpreting. Ambiguous data is recorded as observed, and confirmed before it becomes a "fact".
  • Local-first, explicit egress. Nothing leaves your machine without a deliberate yes; inbound and outbound are never fused.
  • Generic until you personalise it. This blueprint ships anonymized — your categories, entities, and examples come from you, at build time.
